# External Build Fix: flux-lucid

**Date:** 2026-05-17
**Auditor:** Oracle1 (subagent)

## Issue

`Cargo.toml` had a path dependency requiring a sibling checkout:

```toml
spectral-conservation = { path = "../spectral-conservation" }
```

This prevented external developers from building `flux-lucid` unless they also cloned `spectral-conservation` into the correct relative path.

## Resolution

Replaced the path dependency with a crates.io version dependency:

```toml
spectral-conservation = "0.1"
```

`spectral-conservation` v0.1.0 was already published to crates.io (2026-05-17), so this was a one-line fix.

## Build Verification

- `cargo fetch` — downloads cleanly from crates.io registry
- `cargo build` — succeeds with only pre-existing warnings (4 unused-import/variable warnings, no new ones)
- `Cargo.lock` now resolves `spectral-conservation` from `registry+https://github.com/rust-lang/crates.io-index`

## Remaining Issues (low priority)

1. **4 unused imports/variables warnings** — none blocking, easy cleanup via `cargo fix --lib -p flux-lucid`:
